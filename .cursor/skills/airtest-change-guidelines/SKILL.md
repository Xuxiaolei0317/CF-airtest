# Airtest Change Guidelines

## 核心要求

每次新增一些功能或者逻辑的时候添加中文注释

更新项目说明文档

## 执行准则

1. 新增功能、测试流程、状态机节点、业务判断或通用工具方法时，为关键意图、流程分支和不直观的实现添加简洁中文注释。
2. 不为一眼能看懂的赋值、调用、导入添加机械注释，避免注释噪音。
3. 变更影响运行方式、目录结构、脚本命名、设备配置、依赖、测试入口或公共初始化方式时，同步更新 `README.md` 或相关说明文档。
4. 如果本次变更不需要更新文档，在最终回复中简要说明原因。
5. 完成代码或文档修改后，检查最近编辑文件的 linter 诊断，并修复由本次变更引入的问题。
